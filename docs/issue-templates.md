# Issue Templates

Issue templates live in `.github/ISSUE_TEMPLATE`.

## Templates

- `bounty_task.md` is for scoped bounty tasks. It includes bounty labels, acceptance criteria, and a bounty amount section.
- `task.md` is for small maintenance tasks that do not need bounty-specific fields.
- `pi_challenge.md` is for the PI challenge task flow.

## Contributor Guidance

- Use the bounty task template when a reward-backed issue needs a clear scope and acceptance criteria.
- Use the small task template for simple cleanup or documentation work.
- Keep issue titles specific enough that pull requests can clearly reference the task they close.
