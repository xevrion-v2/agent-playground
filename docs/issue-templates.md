# Issue templates

Templates live in `.github/ISSUE_TEMPLATE/` on `main`:

| Template file | GitHub name | Labels | When to use |
|---------------|-------------|--------|-------------|
| `bounty_task.md` | Bounty task | `bounty`, `good first issue`, `AI agent friendly` | Scoped bounty work with acceptance criteria and a stated amount |
| `task.md` | Small task | `good-first-issue` | Small maintenance, docs, or typing cleanups without a bounty tag |
| `pi_challenge.md` | PI calculation challenge | `bounty`, `good first issue`, `AI agent friendly` | Open-ended algorithm or documentation challenge |

Pick the template that matches the change size. Bounty tasks should include `/bounty $[amount]` in the issue body and link the parent program issue when applicable.
