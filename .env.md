# Environment Variables

This document describes the environment variables used across the TaskFlow monorepo. Each app/package under `apps/` and `packages/` expects its own `.env` values for database connectivity, authentication, integrations, and server configuration.

> **Note**: `.env`, `.env.local`, and `.env.*.local` files are gitignored. Create them in the root or inside the respective app/package directory as needed.

---

## Variable Reference

### `PORT`

| Attribute | Value |
|-----------|-------|
| **Used by** | `apps/api` |
| **Required** | No |
| **Default** | `4000` |
| **Description** | TCP port on which the Express API server listens for incoming HTTP requests. |

```env
PORT=4000
```

---

### `DATABASE_URL`

| Attribute | Value |
|-----------|-------|
| **Used by** | `packages/db` (Prisma) |
| **Required** | Yes (for database operations) |
| **Default** | — |
| **Description** | PostgreSQL connection string used by Prisma to connect to the database. Must be a valid Postgres connection URI. |

```env
DATABASE_URL="postgresql://user:password@localhost:5432/taskflow?schema=public"
```

---

### `NEXT_PUBLIC_API_URL`

| Attribute | Value |
|-----------|-------|
| **Used by** | `apps/web` (Next.js) |
| **Required** | No (stub only) |
| **Default** | `http://localhost:4000` |
| **Description** | Base URL of the Express API server, consumed by the Next.js frontend at runtime. Prefixed with `NEXT_PUBLIC_` so Next.js inlines it into the client bundle. |

```env
NEXT_PUBLIC_API_URL=http://localhost:4000
```

---

### `JWT_SECRET`

| Attribute | Value |
|-----------|-------|
| **Used by** | `apps/api` |
| **Required** | Yes (for auth) |
| **Default** | — |
| **Description** | Secret key used to sign and verify JSON Web Tokens (access tokens) for authenticated API requests. Generate a strong random string in production (e.g. `openssl rand -hex 64`). |

```env
JWT_SECRET="your-secret-key-here"
```

---

### `JWT_REFRESH_SECRET`

| Attribute | Value |
|-----------|-------|
| **Used by** | `apps/api` |
| **Required** | Yes (for auth) |
| **Default** | — |
| **Description** | Secret key used to sign and verify JWT refresh tokens. Should be a different value from `JWT_SECRET`. |

```env
JWT_REFRESH_SECRET="your-refresh-secret-key-here"
```

---

### `STRIPE_SECRET_KEY`

| Attribute | Value |
|-----------|-------|
| **Used by** | `apps/api` |
| **Required** | No (Stripe integration is a service placeholder) |
| **Default** | — |
| **Description** | Stripe API secret key (`sk_test_...` or `sk_live_...`) for processing payments server-side. |

```env
STRIPE_SECRET_KEY="sk_test_..."
```

---

### `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY`

| Attribute | Value |
|-----------|-------|
| **Used by** | `apps/web` (Next.js) |
| **Required** | No (Stripe integration is a service placeholder) |
| **Default** | — |
| **Description** | Stripe publishable key (`pk_test_...` or `pk_live_...`) used client-side for Stripe Elements and Checkout. |

```env
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY="pk_test_..."
```

---

### `NODE_ENV`

| Attribute | Value |
|-----------|-------|
| **Used by** | `apps/web`, `apps/api` |
| **Required** | No |
| **Default** | `development` |
| **Description** | Runtime environment mode. Set to `production` in deployed environments, `test` during automated tests. Controls middleware behaviour (error detail verbosity, caching, etc.). |

```env
NODE_ENV=development
```

---

## Quick Start (Local Development)

Create a `.env` file in the repository root (or in `apps/api/` and `apps/web/` separately) with the following minimum set:

```env
# API
PORT=4000
DATABASE_URL="postgresql://postgres:password@localhost:5432/taskflow"
JWT_SECRET="dev-jwt-secret-change-in-production"
JWT_REFRESH_SECRET="dev-refresh-secret-change-in-production"

# Frontend
NEXT_PUBLIC_API_URL=http://localhost:4000

# Environment
NODE_ENV=development
```

## Production Checklist

| Variable | Why It Matters |
|----------|----------------|
| `DATABASE_URL` | Must point to your production Postgres instance. |
| `JWT_SECRET` / `JWT_REFRESH_SECRET` | Use strong random secrets (e.g. `openssl rand -hex 64`). |
| `STRIPE_SECRET_KEY` | Use live keys (`sk_live_...`), never test keys. |
| `NODE_ENV` | Must be `production`. |
